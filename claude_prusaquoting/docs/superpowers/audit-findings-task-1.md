| 1.1a | markup=0 produces None quote_price | Bug | High | 1087 | `unit_price = subtotal * (1 + float(markup) / 100) if markup and subtotal else None`. When `markup` is the string `"0"`, Python treats it as truthy (non-empty string), so this actually works. However, when `markup` is the empty string `""` (user leaves field blank), `unit_price` becomes `None` and the quote shows no price. This is arguably correct ("no markup entered = no price"), but the conditional conflates "not provided" with "zero". | Change the guard to `if markup is not None and markup != "" and subtotal` or default markup to `"0"` when empty. |
| 1.1b | qty_subtotal=0 suppresses total_price | Bug | Medium | 1107 | `total_price = qty_subtotal * (1 + float(markup) / 100) if markup and qty_subtotal else None`.
