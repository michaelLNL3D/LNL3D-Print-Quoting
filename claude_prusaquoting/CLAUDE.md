# 3D Print Quoting Tool — CLAUDE.md

## What this project is
A local Flask web app (`app.py`) that accepts 3D model files (STL, OBJ, 3MF, AMF, STEP/STP), slices them via PrusaSlicer CLI, and produces a detailed quote including print time, filament weight, material cost, machine cost, markup, and a final price. `quote.py` is a standalone CLI version of the same logic.

**URL:** http://localhost:5111
**Start:** `python3 app.py` (opens browser automatically)
**Port:** 5111 (override with `PORT=` env var)

---

## Key architecture

### Single file
All backend routes, business logic, HTML, CSS, and JavaScript live in `app.py` (~2850 lines). `render_template_string` is used — HTML/JS changes take effect on page reload; Python route changes require a server restart.

### Slicer paths (top of app.py)
```python
PRUSASLICER  = "/Applications/PrusaSlicer.app/Contents/MacOS/PrusaSlicer"
ORCASLICER   = "/Applications/OrcaSlicer.app/Contents/MacOS/OrcaSlicer"
PRUSA_VENDOR_DIR = "/Applications/PrusaSlicer.app/Contents/Resources/profiles"
PRUSA_USER_DIR   = "~/Library/Application Support/PrusaSlicer"
```
Override via environment variables: `PRUSASLICER_PATH`, `ORCASLICER_PATH`, `PRUSA_VENDOR_DIR`, `PRUSA_USER_DIR`.

### Temp/export storage
- `EXPORT_DIR = /tmp/prusaquoting_exports/` — persists between requests for GCode download
- GCode files named `{job_id}_piece_{idx}.gcode` — job_id generated before slicing loop to prevent collisions in batch mode
- In-memory `_jobs` dict (last 50 jobs): `job_id -> {gcode_paths, filename}`
- In-memory `_last_job` dict: stores state for 3MF/GCode export
- In-memory `_progress` dict: `pid -> list of {label, status, detail}` — live progress steps for active quotes. No size cap (unlike `_jobs`); JS cleans up via DELETE after completion but errors/crashes can leak entries.
- In-memory `_error_log` list (last 100 entries): logged by `_log_error()`, exposed via `/api/debug_log`
- `_emit(pid, label, status, detail)` — helper to push/update a progress step; no-ops when `pid` is empty

### Presets
Saved to `~/.prusaquoting_presets.json`. Keys: `printer`, `print_profile`, `filament`, `infill`, `layer_height`, `walls`, `supports`, `size_mode`, `size_val`, `cost_per_kg`, `hourly_rate`, `markup`, `time_factor`, `farm_size`.

## Slicing flow

1. **Pre-flight** (`/api/check_size`): Uses `--info` flag (no slicing) to get bounding box via `get_mesh_bounds()`. Compares to build volume from INI inheritance chain. Returns `overflow_warning` if oversized.
2. **User chooses** Scale to Fit / Auto-Split / Quote Anyway per file.
3. **`/api/quote`**:
   - Optionally auto-orients via OrcaSlicer CLI
   - Optionally splits via Meshmixer/PrusaSlicer (if `auto_split=true`)
   - Optionally scales to fit via `--scale-to-fit X,Y,Z` (95% of build volume)
   - Slices each piece via `slice_piece()` → `--export-gcode`
   - Parses time/weight/cost from GCode comments
   - Applies markup, quantity, farm size multipliers
   - Emits live progress steps via `_emit()` using a `progress_id` (UUID generated by JS, passed as `progress_id` form field); JS polls `/api/progress/<pid>` while slicing

### Build volume detection
Reads `bed_shape` and `max_print_height` from PrusaSlicer INI files, walking the `inherits` chain up to 8 levels deep. User printer profiles in `~/Library/Application Support/PrusaSlicer/printer/` take priority.

## JavaScript patterns

### File queue (`fileQueue[]`)
Each item: `{ file, status, settings, result, sizeCheck, sizeOverride }`.
- `status`: `'pending'` | `'done'` | `'error'`
- `sizeOverride`: `'scale_fit'` | `'split'` | `'warn'` | `undefined`

### Key JS functions
- `runBatch()` — pre-flight check then calls `_runBatchSlice()`
- `_runBatchSlice()` — iterates queue, calls `/api/quote`, updates UI
- `buildFormData(file, settings, sizeOverride)` — assembles FormData for `/api/quote`
- `showResults(d)` — renders single-file quote result
- `showBatchResults()` — renders batch summary table
- `openQuoteModal(i)` — popup with individual quote details (batch mode)
- `copyQuote()` — copies formatted quote text to clipboard
- `confirmPreflight()` — user confirmed oversized file action, resumes slicing

### Known JS quirks
- Python `'\n\n'` inside a triple-quoted HTML string becomes a literal newline, breaking JS string literals. Always use `'\\n\\n'` when embedding newlines in JS strings within Python HTML.
- `parseTimeToMins(t)` in the JS (line ~2257) handles `d/h/m/s` — must include `d` in regex.

## Cost calculation
```
material_cost = (weight_g / 1000) * cost_per_kg
machine_cost  = (print_time_mins / 60) * hourly_rate
subtotal      = material_cost + machine_cost
quote_price   = subtotal * (1 + markup / 100)
```
Farm mode: time divided by farm_size (parallel printing), weight/cost multiplied by quantity.

## Docker
`Dockerfile` + `docker-compose.yml` available. Requires `webkit2gtk` for PrusaSlicer on Linux. Bind address is `0.0.0.0` in container. Config paths need to be volume-mounted.

## External quoting integration
The `/api/quote` response contains all data needed to push to an external quoting program. See the "quote_export" integration section — the app can POST quote data as JSON to a configurable webhook URL, or export as CSV. Configure the webhook URL in the settings panel or via the `QUOTE_WEBHOOK_URL` environment variable.
